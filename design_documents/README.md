## 🎯 为什么需要这个目录？

### 问题
切换Claude对话后，可能丢失设计决策和上下文：
- 不知道为什么这样设计
- 不知道迭代历程
- 不知道用户的真实需求

### 解决方案
将关键设计决策记录在设计文档中：
- 所有Claude都可以读取
- Git版本控制
- 可追溯的设计演进

---

## 📖 如何使用

### 对于新启动的Claude

**必读**（General和Planner）：
- 首次启动时必须阅读对应的设计文档
- 理解系统架构和设计理念
- 理解自己的角色定位

**建议阅读**（Assessor、Teacher、Researcher）：
- 了解系统架构
- 理解协作机制
- 更好地履行职责

### 对于用户

- 查看设计决策的完整记录
- 理解系统设计理念
- 验收系统时参考
